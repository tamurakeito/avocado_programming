第0章では、なぜ私たちがアプリ開発の第一歩として Flutter を選ぶのか、そしてこの講座で何を作っていくのかをお話ししました。

この章では、いよいよ Flutter そのものに少しだけ触れていきましょう。
まだ皆さんのパソコンで動かす準備はできていませんが、まずは「Flutter のコードって、どんな感じでアプリの見た目になるの？」という雰囲気を掴んでもらうのがゴールです。

## Flutterアプリのコードを覗いてみよう
プログラミングと聞くと、黒い画面に謎の英語がたくさん...というイメージがあるかもしれません。実際のアプリのコードはもっと複雑ですが、ここでは特別にシンプルな例を見てみましょう。

これは、画面の真ん中に「Hello, World!」と表示するだけの、とても簡単な Flutter アプリのコードの一部です。（Playground で実行してみてください！）

<CodeBlock language="dart" gist="ea9a498e7db775c51e650d1be4bb0c3f">
  {`import 'package:flutter/material.dart'; // Flutterのお約束ごとを読み込む

  void main() { // アプリケーションはここから始まる
    runApp(MyApp()); // MyAppというアプリを起動する
  }

  class MyApp extends StatelessWidget { // アプリ全体の設計図
    @override
    Widget build(BuildContext context) {
      return MaterialApp( // アプリの基本的な見た目やルールを設定
        home: Scaffold( // 画面の骨組み（土台）
          appBar: AppBar( // 画面上部のバー（タイトルバー）
            title: Text('はじめてのアプリ'), // バーに表示する文字
          ),
          body: Center( // 画面の中央に配置する
            child: Text('Hello, World!'), // 表示したい文字
          ),
        ),
      );
    }
  }`}
</CodeBlock>

「うわあ...英語ばっかり...」と思った方もいるかもしれません。大丈夫！ 今は一行一行の意味を完璧に理解する必要は全くありません。

ここで感じてほしいのは、`AppBar` や `Text` といった **「部品の名前」** のようなものがコードの中に書かれていて、 **それが実際のアプリ画面の「タイトルバー」や「表示される文字」に対応しているらしい** 、という雰囲気です。

- `AppBar(...)` と書くと、画面の上にタイトルバーが表示されるんだな。
- `Text('Hello, World!')` と書くと、その文字が画面に出るんだな。
- `Center(...)` で囲むと、真ん中に来るのかな？

こんな風に、コードが **「画面の設計図」** になっているようなイメージを持てれば十分です。

## 画面の見た目は「宣言」するもの

Flutter の面白い特徴の一つに、 **「宣言的（せんげんてき）UI」** という考え方があります。...といっても難しく考えなくて大丈夫です！

なんだか難しそうに聞こえますが、これは「カーナビの目的地設定」と「口頭での道案内」の違いに例えると分かりやすいかもしれません。

これまでのプログラミング（これを **命令的（めいれいてき）** と言います）では、まるで隣に座っている人に口頭で道を案内するように、一つ一つの手順を細かく指示する必要がありました。
「まず、この道を200メートル直進してください。次の信号を右に曲がって、3つ目の角を左です... 」といった具合です。UIを作る際も、「このボタンを画面のこの位置に置いて、色を青に変えて、文字を白にして... 」と、一つ一つの操作を命令していました。

一方、Flutter の宣言的UIは、カーナビに「最終目的地は恵比寿駅です」と設定するようなものです。
私たちは **「最終的にどうなっていてほしいか」というゴール（完成図）** を宣言するだけです。

そうすれば、カーナビ（Flutter）が現在地から目的地までの最適なルート（UIの描画方法や更新手順）を自動で計算し、私たちを連れて行ってくれます。
私たちがいちいち「次の道を右だ！」と指示する必要はありません。

Flutter のコードで `Text('こんにちは')` と書くのは、まさに「画面に『こんにちは』というテキストが表示されている状態がゴールです」と宣言しているのと同じなのです。
Flutter がその宣言を読み取り、どうやってそれを画面に表示するかという面倒な部分は、すべて裏側で賢く処理してくれます。

これは、「どうやって（How）」を細かく指示するよりも、「何を（What）」表示したいかを記述する方が、特に画面の見た目を作る上では直感的でわかりやすいことが多い、という考え方に基づいています。

## コード変更がすぐ反映される！「ホットリロード」

もう一つ、Flutter を学ぶ上で知っておくとワクワクする機能があります。それが「ホットリロード」と「ホットリスタート」です。

これは、**書いたコードの変更が、開発中のアプリ画面にすぐに反映される**機能です。

### ホットリロード (Hot Reload)

 例えば、先ほどのコードの `Text('Hello, World!')` を `Text('こんにちは、世界！')` に書き換えたとします。
 ホットリロードを使うと、開発中のスマホ画面（シミュレータや実機）で、**アプリの状態を保ったまま、一瞬で文字だけが「こんにちは、世界！」に切り替わる**んです。

これは、レゴブロックで作っている最中に、赤いブロックを一つだけ青いブロックに取り替えるような感覚です。全体を壊さずに、変更したい部分だけサッと変えられます。
文字の色を変えたり、部品の配置を少しずらしたり...といった見た目の調整をするときに、ものすごく便利で、開発が楽しくなります。

### ホットリスタート (Hot Restart)

こちらは、ホットリロードよりも少しだけアプリの内部構造に変更があった場合に使うもので、アプリ全体を素早く再起動するイメージです。
それでも、通常のアプリ開発で「コード修正 → 再ビルド → 起動」という時間のかかるプロセスに比べると、ずっと高速です。

これらの機能のおかげで、Flutter 開発では「試してみる → すぐ確認する → また試す」というサイクルを非常に速く回すことができます。
これは、特にプログラミングを学び始めたばかりの皆さんにとって、試行錯誤しやすく、モチベーションを保つ上で大事な機能なのです！
